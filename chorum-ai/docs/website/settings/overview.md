# Settings Overview

Configure Chorum to work your way.

---

## Accessing Settings

Click the **gear icon** in the sidebar, or go to `/settings` directly.

![Settings page](../images/providerstab.jpg)

---

## Settings Sections

The sidebar shows all settings categories:

| Section | What It Does |
|---------|--------------|
| **Providers** | Add API keys, set budgets, manage models |
| **General** | Profile info, display preferences |
| **Security** | HTTPS enforcement, PII anonymization, SSL |
| **Memory & Learning** | Auto-learn, context injection, summarization |
| **Web Search** | Configure Serper API, enable internet access |
| **MCP Integration** | Connect external AI agents (Claude Desktop, Cursor) |
| **Resilience** | Fallback chains, local model configuration |
| **Help** | Documentation links, support |
| **Legal & Privacy** | Terms, privacy policy |
| **About** | Version info, credits |

---

## Quick Stats

The Providers page shows at a glance:
- **Total Usage Today**: How much you've spent
- **Total Daily Budget**: Your combined limits
- **Per-Provider Status**: Which are active, usage, remaining budget

---

## Key Settings to Configure

### 1. Providers (Start Here)
Add at least one API key to start chatting. See [Adding API Keys](./api-keys.md).

### 2. Budgets
Set daily limits per provider to avoid surprise bills. See [Budget Management](./budgets.md).

### 3. Memory & Learning
Enable auto-learning so Chorum remembers your patterns and decisions.

### 4. Web Search
Enable internet access for real-time information. See [Web Search](./web-search.md).

### 5. Security
Enable PII anonymization if you're working with sensitive data.
