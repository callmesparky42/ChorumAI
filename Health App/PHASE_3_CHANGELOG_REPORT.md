## Decision Log (Including Rewrite/Overwrite Choices)

1. Rewrote Garmin credential IV storage format in connect route.
- Decision: store `credsIv` as `usernameIv:passwordIv`.
- Reason: current crypto decrypt signature uses `decryptPHI(ciphertext, iv)` and expects IV-only handling.
- Change made: removed tag-pair packing format (`userIv:userTag|passIv:passTag`) and replaced with IV-only format.

2. Rewrote Garmin credential decrypt logic in sync core.
- Decision: remove legacy 3-argument decrypt usage and parse only two IVs from `credsIv`.
- Reason: aligns with current crypto module behavior and Phase 3 spec intent.
- Change made: `decryptPHI(..., iv)` only; removed tag parsing and legacy pair splitting.

3. Rewrote Garmin snapshot IV persistence format.
- Decision: write `payloadIv: encrypted.iv` for new Garmin snapshots.
- Reason: keep new writes in Phase 3-compatible IV format while crypto still tolerates legacy `iv:tag` reads.
- Compatibility choice: did not migrate historical rows; read-path compatibility remains via existing crypto handling.

4. Rewrote weekly health-checkup cron flow (`/api/cron/health-checkup`).
- Decision: replace temporal/stub-adjacent behavior with de-identified payload analysis flow and alert learning creation.
- Changes made:
  - kept cron secret protection via `Authorization: Bearer`.
  - decrypt + `deidentifyObject` before provider call.
  - provider selection now uses existing `selectHealthProvider(...)`.
  - anomaly heuristic retained and simplified.
  - creates learning rows when anomalies are found (`type: health_alert`).
  - stores `checkup_result` snapshot with encrypted payload.
- Overwrite/rewrite note: this was a substantive route rewrite, not a partial edit.

5. Rewrote MCP `health_checkup` handler to Phase 3 contract.
- Decision: remove Phase 2 summary/stub output shape and return MCP enrichment payload.
- New response shape now includes:
  - `snapshotsAnalyzed`
  - `periodDays`
  - `deidentifiedData`
  - `instruction`
- PHI audit action kept and aligned to snapshot viewing.
- Overwrite/rewrite note: old Phase 2 stub behavior was intentionally replaced.

6. Rewrote `health-handlers` tests for the new `health_checkup` contract.
- Decision: update assertions to match Phase 3 behavior.
- Removed obsolete expectations about excluded text-note stub output.
- Added assertions for `deidentifiedData`, `instruction`, and snapshot audit log semantics.

7. Corrected Garmin dependency version to a publishable version.
- Initial state: `garmin-connect@^1.8.1` failed install (`ETARGET` no matching version).
